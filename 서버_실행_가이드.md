# 🚀 서버 실행 가이드

## 현재 상태 확인

❌ **서버가 실행되지 않았습니다.**

확인 사항:
- ❌ `node_modules` 폴더 없음 → `npm install` 필요
- ⚠️ `.env.local` 파일 없음 → API 키 설정 (선택사항)
- ❌ 개발 서버 미실행

---

## 실행 방법 (단계별)

### 1단계: PowerShell 실행 정책 확인

PowerShell에서 다음 명령어 실행:

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

또는 `npm.cmd` 사용:
```powershell
npm.cmd install
npm.cmd run dev
```

---

### 2단계: 의존성 설치

프로젝트 디렉토리에서:

```powershell
cd c:\moviessam2\mvdebate
npm install
```

**예상 소요 시간**: 1-3분  
**예상 출력**:
```
added 500 packages, and audited 501 packages in 2m
found 0 vulnerabilities
```

---

### 3단계: 환경 변수 설정 (선택사항)

`.env.local` 파일을 프로젝트 루트에 생성:

```env
UPSTAGE_API_KEY=your_upstage_api_key_here
```

**참고**: API 키가 없어도 서버는 실행되지만, AI 토론 기능은 작동하지 않습니다.

---

### 4단계: 개발 서버 실행

```powershell
npm run dev
```

**예상 출력**:
```
> mvdebate@1.0.0 dev
> next dev

  ▲ Next.js 14.0.0
  - Local:        http://localhost:3000
  - Ready in 2.3s
```

**중요**: 서버가 실행되면 터미널 창을 닫지 마세요!

---

### 5단계: 브라우저에서 확인

서버가 실행되면 브라우저에서 다음 주소로 접속:

```
http://localhost:3000
```

---

# 3. 의존성 설치 (처음 한 번만)

## 문제 해결

### "npm이 인식되지 않습니다"

**해결 1**: `npm.cmd` 사용
```powershell
npm.cmd install
npm.cmd run dev
```

**해결 2**: 실행 정책 변경
```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

**해결 3**: CMD 사용
- VS Code 터미널에서 "명령 프롬프트" 선택
- 또는 `Win + R` → `cmd` 입력

---

### "포트 3000이 이미 사용 중입니다"

다른 포트로 실행:
```powershell
$env:PORT=3001; npm run dev
```

브라우저에서 `http://localhost:3001` 접속

---

### "UPSTAGE_API_KEY is not set"

이 오류는 무시해도 됩니다. 서버는 정상 실행되지만 AI 기능만 작동하지 않습니다.

API 키를 설정하려면 `.env.local` 파일 생성:
```env
UPSTAGE_API_KEY=your_api_key_here
```
